### Added
- **Data Contract: Story-Facing vs System-Facing v0.1** - Established explicit, versioned contract defining which campaign/session data fields serve narrative purposes (GM/player readable) versus system purposes (internal metrics, reproducibility). Story-facing data includes canon_summary, what_happened, manual_entries (title/description/notes), and session_notes. System-facing data includes severity, cutoff_rate, tags, deltas (pressure/heat changes), source metadata, prep_item_ids, and determinism keys (session_id). Contract establishes six core rules: (1) Ledger stores both, (2) Default exports choose story lens, (3) Editing happens story-side, (4) System data never required for story, (5) Story data never required for system, (6) System-facing never leaks into narrative exports. Includes complete field classification tables, migration notes, implementation guidance, and "read-aloud test" principle. Foundation for future export redesign and web story view. See `docs/DATA_CONTRACT_story_vs_system_v0.1.md` for complete contract specification.
